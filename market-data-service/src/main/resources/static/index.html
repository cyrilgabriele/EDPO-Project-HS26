Event Notification

The Event Notification pattern decouples the producer from all consumers. This service publishes a CryptoPriceUpdatedEvent for each symbol every poll cycle, then forgets about it.

Any number of downstream services can subscribe to crypto.price.raw without requiring any change to this service — the producer is completely unaware of who (or how many) consumers exist.

▶ Symbol used as Kafka message key → partition affinity guarantees per-symbol ordering across all 3 partitions.
▶ acks=all configured: the broker only acknowledges once all in-sync replicas have written the message.
✓ Polling interval: 10 s (configurable via BINANCE_POLL_INTERVAL_MS).
