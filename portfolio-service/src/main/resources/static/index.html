Event-Carried State Transfer

In the ECST pattern, each event carries enough data for the consumer to update its own local state — no callback to the producer is needed.

This service receives CryptoPriceUpdatedEvent messages from Kafka and writes the price directly into its LocalPriceCache. REST queries are answered entirely from that local replica.

▶ Autonomy: can serve price and portfolio REST requests even when market-data-service is temporarily offline.
✓ No synchronous coupling: zero HTTP calls from this service to the producer at query time.
⚠ Eventual consistency: cached prices lag by up to one poll interval (default: 10 s).
